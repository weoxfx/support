=====================================
1. OVERVIEW
=====================================

XeoWallet is a digital wallet web application that enables instant peer-to-peer money transfers using XIDs (unique user identifiers). Users can send, receive, add funds, withdraw, create Lifafas (digital envelopes/scratch cards), play games, and use a programmatic payment API.

- Currency: Indian Rupees (₹ / INR)
- Website: https://xeowallet.vercel.app
- Target: Indian users, Telegram community-driven

=====================================
3. USER IDENTITY
=====================================

Every user has:
- XID: A unique wallet identifier (e.g., "XEO123") used to send/receive money
- Username: Display name
- Mobile: Phone number (required at signup)
- Name: Optional display name
- Balance: Current wallet balance in ₹
- Telegram ID: Optional, used for webhook notifications
- Avatar URL: Optional profile picture

Users are identified by their `user_id` (Supabase auth UUID) internally, but externally by their XID.

=====================================
4. AUTHENTICATION
=====================================

- Full Name, Username, Mobile, Password
- Username is Unique

=====================================
5. CORE FEATURES
=====================================

5.1 DASHBOARD
- Shows current balance
- Recent transactions list
- Quick action buttons (Send, Add Funds, Withdraw)
- Real-time balance updates via Supabase Realtime

5.2 SEND MONEY
- Send ₹ to any user by their XID
- Optional comment/note
- Instant transfer (balance deducted from sender, added to recipient)
- Creates transaction records for both sender and recipient
- Creates notification for recipient
- Sends Telegram webhook if user has telegram_id configured

5.3 ADD FUNDS
- User submits a fund request with:
  - Amount
  - UPI screenshot upload
  - UTR number
- Request goes to "pending" status
- 1% Bonus
- Approved/rejected by admin (balance credited on approval)

5.4 WITHDRAW
- User submits withdrawal request with:
  - Amount
  - UPI ID
- Request goes to "pending" status
- Tax 3%
- Approved/rejected by admin (balance deducted on approval)

5.5 LIFAFA (Digital Envelopes)
- Creator sets: total amount, max winners, amount per user
- Generates a unique code
- Types: normal scratch card, coin toss, dice roll
- Optional password protection
- Optional required Telegram channel membership check
- Users claim by entering the code
- IP-based duplicate prevention
- When all winners claimed or manually deactivated, lifafa closes

5.6 GAMES — Big or Small
- Real-time multiplayer betting game
- 10-second betting rounds
- Users pick "Big" or "Small" and place bets (₹5–₹1000)
- Result revealed after round ends
- Win payout: 1.25x bet amount
- Realtime presence tracking (online user count)
- Realtime bet updates

5.7 NOTIFICATIONS
- In-app notification system
- Types: success, info, warning, error
- Marked as read/unread
- Real-time delivery

5.8 TRANSACTIONS
- Full transaction history page
- Types: send, receive, add_funds, withdraw, lifafa
- Status: pending, completed, rejected
- Filterable and sortable

5.9 PROFILE
- View/edit username, name, avatar, telegram_id
- View XID and balance
- QR code for receiving payments

=====================================
6. API SYSTEM
=====================================

XeoWallet has a programmatic payment API for automated transfers.

6.1 API KEY MANAGEMENT (User-facing page: /api)
- Users can generate/regenerate their API key
- Keys can be activated/deactivated
- One key per user

Parameters:
  - KEY (required): User's API key
  - XID (required): Recipient's XID
  - AMOUNT (required): Amount in ₹ (positive number)
  - COMMENT (optional): Payment note

Response (success):
{
  "success": true,
  "data": {
    "transaction_id": "uuid",
    "status": "completed",
    "amount": 100,
    "xid": "recipient_xid",
    "comment": "optional note",
    "timestamp": "ISO date"
  }
}

Response (error):
{
  "success": false,
  "error": "Error message"
}

Validations:
- API key must exist and be active
- Sender must have sufficient balance
- Sender/recipient must not be frozen or banned
- Cannot send to self
- Amount must be positive

Flow:
1. Validate API key → get sender user_id
2. Get sender profile, check balance/status
3. Get recipient by XID, check status
4. Deduct from sender, add to recipient
5. Create transaction records for both
6. Create notification for recipient
7. Send Telegram webhooks if configured
